# Проект «Многоцелевая модель для NER и классификации»

## Краткое описание

В рамках проекта рассматривается задача одновременного извлечения именованных сущностей (NER, токен-уровень) и классификации событий/отношений (документ-уровень, multihot) на основе русскоязычного датасета NEREL.

**Проблема:** традиционный пайплайн выполняет NER и классификацию событий раздельно, что ведёт к дублированию признаков и неэффективному расходу вычислительных ресурсов.

**Гипотеза:** совместное (multi‑task) обучение на одном энкодере позволяет повысить качество NER и стабильность детекции событий за счёт взаимного влияния задач.

**Цель проекта** — построить компактную воспроизводимую мультизадачную модель на основе трансформерного энкодера, которая:
- решает обе задачи (NER и классификацию событий) в одном прямом проходе;
- обеспечивает количественную (token‑F1, micro‑F1) и качественную (анализ ошибок) оценку взаимного влияния задач;
- позволяет сравнить эффективность (память, скорость, точность) с раздельным подходом и квантизованной версией.

## Данные

Используйте публичный датасет **NEREL** в формате JSONL: `train`, `dev`, `test`. Этот датасет содержит тексты, сущности (строки с офсетами) и отношения/события. Его можно скачать с Hugging Face.

## Основные этапы выполнения

### 1. EDA (разведочный анализ)
- Частотность типов сущностей и отношений.
- Распределение длины текстов и количества сущностей на документ.
- Выводы по структуре датасета.

### 2. Парсинг и подготовка таргетов
- Извлечение сущностей из JSONL → BIO-метки для каждого токена (word-level).
- Построение multihot-вектора для классификации событий (топ‑30 наиболее частых типов).

### 3. Токенизация и выравнивание меток
- Использование `AutoTokenizer` (русскоязычная модель, например `cointegrated/rubert-tiny2` или `DeepPavlov/rubert-base-cased`).
- Выравнивание BIO-меток на субтокены (первый субтокен получает метку, остальные – `-100`).
- Создание `DataLoader` с `DataCollatorForTokenClassification`.

### 4. Модель (JointModel)
- Общий энкодер BERT/RuBERT.
- Две головы:
  - Линейный слой для NER (классификация каждого токена).
  - Линейный слой от `pooled_output` (или `[CLS]`) для multihot классификации.
- Функция потерь: взвешенная сумма `token_loss` (CrossEntropy) и `cls_loss` (BCEWithLogitsLoss). Реализован вариант с **uncertainty weighting** (обучаемые веса для балансировки задач).

### 5. Обучение и валидация
- Оптимизатор AdamW, scheduler (например, линейный с прогревом).
- Gradient clipping.
- Метрики:
  - Token-level F1 (macro) и span-F1 (через `seqeval`).
  - Multihot micro-F1, precision, recall.
- Логирование метрик по эпохам, сохранение лучшей модели.

### 6. Инференс и качественный анализ
- Функция предсказания для произвольного текста → токены, BIO-теги, вероятности классов событий.
- Анализ ошибок на 8–10 примерах.
- Квантизация (динамическая PTQ) для ускорения инференса на CPU, сравнение качества и времени выполнения.

### 7. Выводы
- Сравнение метрик baseline (однозадачная модель) и мультизадачной.
- Оценка влияния совместного обучения на каждую задачу.

## Результаты

**Ожидаемые метрики (проходной минимум):**  
- Token-level F1 (NER) ≥ 0.5  
- Multihot micro-F1 (классификация событий) ≥ 0.8  

**Итоговый отчёт включает:**  
- Jupyter Notebook с полным выполнением.  
- Графики обучения и таблицы метрик.  
- Сравнение исходной и квантизованной моделей (размер, скорость, точность).  
- Выводы о целесообразности совместного обучения.

## Требования к аппаратному обеспечению

- **GPU**: рекомендуется одна NVIDIA GPU с объёмом памяти ≥ 8 ГБ (например, T4).  
- **Время выполнения**: 5–6 часов на одной GPU (зависит от выбранной модели и количества эпох).  
- **RAM**: 16 ГБ.
