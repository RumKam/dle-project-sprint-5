# Проект «Многоцелевая модель для NER и классификации»

## Краткое описание

В рамках проекта рассматривается задача одновременного извлечения именованных сущностей и классификации событий/отношений на основе русскоязычного датасета NEREL.

**Проблема:** традиционный пайплайн выполняет NER и классификацию событий раздельно, что ведёт к дублированию признаков и неэффективному расходу вычислительных ресурсов.

**Гипотеза:** совместное обучение на одном энкодере позволяет повысить качество NER и стабильность детекции событий за счёт взаимного влияния задач.

**Цель проекта** построить компактную воспроизводимую мультизадачную модель на основе трансформерного энкодера, которая:
- решает обе задачи NER и классификацию событий в одном прямом проходе;
- обеспечивает количественную (token‑F1, micro‑F1) и качественную оценку взаимного влияния задач;
- позволяет сравнить эффективность (память, скорость, точность) с раздельным подходом и квантизованной версией.

## Данные

Используется публичный датасет **NEREL** в формате JSONL: `train`, `dev`, `test`. Этот датасет содержит тексты, сущности и отношения/события.

## Основные этапы выполнения

### 1. EDA
- Частотность типов сущностей и отношений.
- Распределение длины текстов и количества сущностей на документ.
- Выводы по структуре датасета.

### 2. Парсинг и подготовка таргетов
- Извлечение сущностей из JSONL → BIO-метки для каждого токена word-level.
- Построение multihot-вектора для классификации событий (топ‑30 наиболее частых типов).

### 3. Токенизация и выравнивание меток
- Использование `AutoTokenizer` (русскоязычная модель `rubert-base-cased`).
- Выравнивание BIO-меток на субтокены.
- Создание `DataLoader` с кастомным `collate_fn`.

### 4. Модель (JointModel)
- Общий энкодер.
- Две головы:
  - Линейный слой для NER, классификация каждого токена.
  - Линейный слой от `pooled_output` для multihot классификации.
- Функция потерь: взвешенная сумма `token_loss` CrossEntropy и `cls_loss` BCEWithLogitsLoss. Реализован вариант с **uncertainty weighting** обучаемые веса для балансировки задач.

### 5. Обучение и валидация
- Оптимизатор AdamW, scheduler линейный с прогревом.
- Gradient clipping.
- Метрики:
  - Token-level macro-F1 (NER).
  - Multihot micro-F1 (классификация событий).
- Логирование метрик по эпохам, сохранение лучшей модели.

### 6. Инференс и качественный анализ
- Функция предсказания для произвольного текста → слова, BIO-теги, вероятности классов событий.
- Анализ ошибок на 10 примерах.
- Квантизация (динамическая PTQ) для ускорения инференса на CPU, сравнение качества и времени выполнения.

### 7. Выводы
- Сравнение метрик однозадачной и мультизадачной моделей.
- Оценка влияния совместного обучения на каждую задачу.

## Результаты

**Итоговые метрики на тестовом датасете (после 15 эпох, `rubert-base-cased`):**

- NER macro-F1 = **0.7302** (P=0.7604, R=0.7213) превышает проходной минимум 0.5.
- CLS micro-F1 = **0.7761** (P=0.8158, R=0.7401) близок к целевому 0.8.

Качественные примеры предсказаний  показывают, что модель правильно выделяет большинство сущностей, хотя иногда нарушает BIO-схему (повторяет `B-` вместо `I-`). Редкие типы, например, `AWARD`, `LOCATION`, распознаются хуже, что связано с дисбалансом классов.
